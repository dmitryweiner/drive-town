# CLAUDE.md — карта проекта для агента

Drive Town: браузерная игра (TypeScript + Vite + Canvas) — машинка едет по
процедурному городу до цели, нарушения ПДД копятся в счётчик, очки за время и
чистоту. Физика/коллизии/знаки/стиль кода взяты из ../driving-trainer (см. его
CLAUDE.md). Спецификация для людей — в README.md.

## Команды

```bash
npm test               # vitest, ~105 тестов — ДОЛЖНЫ быть зелёными после любой правки
npm run lint           # eslint; `as`-касты ЗАПРЕЩЕНЫ (assertionStyle: never)
npx tsc --noEmit       # строгие типы
npm run dev            # Vite на :5173; ?seed=123 — фиксированный город
npm run build          # tsc + сборка в ./docs (GitHub Pages)
```

Процесс — TDD: сначала тест, потом код. UI-тексты — на русском.
Скриншоты: playwright лежит в ../driving-trainer/node_modules (см. историю —
скрипт из scratchpad запускает vite на :5199 и жмёт клавиши).

## Координаты и константы (как в driving-trainer)

- x — на восток, y — на юг (canvas), heading 0 = +x. Все улицы осеориентированы.
- Все дороги 2 полосы: HALF_ROAD=4.5, центр полосы ±LANE_OFF=2.25 от осевой,
  правостороннее движение (право от направления = rightOf = {-y, x}).
- Квадрат перекрёстка — узел ±4.5; стоп-линия за 1 м до квадрата; углы
  скруглены CURB_PAD=1.5 (не убирать — хвост кузова цепляет).
- Ребро: a→b, oneWay = движение только a→b; crosswalks/спавны меряются
  метрами ОТ ЦЕНТРА узла a.
- Повороты — дуги: правый R=4.5, левый R=5.5; turnPath может начинаться/
  кончаться чуть ЗА квадратом (точки касания дуг лежат на осях полос).
- Светофор: цикл 20 с (8 зел + 2 жёлт + 9 красн + 1 кр-жёлт); вертикальная
  группа (подъезды N/S) зелёная в начале цикла; сдвиг — node.lightOffset.

## Ключевые решения (не ломать)

- **Нарушение ≠ провал.** Всё копится в счётчик (выбор пользователя). Одно
  событие на эпизод: латчи в RuleMonitor (speeding/wrong-way/off-road/контакты).
- **Жёлтый прощается**, если при его включении остановиться было нельзя
  (эвристика: elapsed×v > v²/8+1 → нарушение). Автопилот и NPC при зелёном
  предсказывают фазу к моменту прибытия.
- **Спавн игрока — на ПЕРВОМ ребре маршрута курсом от spawnNode**: маршрут
  никогда не требует разворота (разворот игрок физически не вписывает в
  9-метровую дорогу одним движением). Цель — не на ребре прибытия и с
  легальным выездом (canTravel).
- **NodeArbiter**: бронь узла берёт только ПЕРВЫЙ в очереди (blockedAhead),
  иначе задний захватывает бронь и колонна виснет намертво (был баг).
- **clearToEnter** игнорирует стоящих (|v|<0.5) вне квадрата — иначе гридлок
  «все ждут всех» на нерегулируемом; занятый квадрат блокирует всех.
  Левый поворот/разворот добавляет встречную сторону в yieldSides.
- **Помеха оценивается по времени прибытия** (tta < 4 с и ближе 30 м) — по
  дистанции NPC успевал «нырнуть» под игрока.
- **Пешеходы: 1 на зебру** с паузами 5–11 с — при двух поток почти не находит
  окна и трафик стоит перед переходом минутами.
- NPC в тупике разворачивается по deadEndLoop (полукруг R=2.25) — годится
  только для NPC-путеследователей, НЕ для машины игрока.
- Прямой проезд узла НЕ ограничен TURN_SPEED (только дуги).

## Тесты

- tests/completable.test.ts — сквозной: автопилот проходит 8 сидов с трафиком
  без нарушений. При правках движка гонять свип по 30 сидам (см. историю).
- Геометрические константы (HALF_ROAD, радиусы дуг, LIGHT_CYCLE) завязаны на
  citymap/rules/traffic-тесты и автопилот — не менять без полного прогона.
- Не давать rules-тестам «идеальные» ожидания к жёлтому: правило прощения.

## Чего не делать

- Не коммитить без просьбы пользователя.
- Не менять смысл dirSign (1 = a→b) и порядок вершин ребра — на нём завязаны
  oneWay, crosswalks.at и lanePoint.
- Не выпиливать «твёрдость» домов (Round.hitsSolid) — off-road остаётся
  нарушением, но сквозь дома ездить нельзя.
