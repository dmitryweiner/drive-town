# SOUND.md — дизайн звука (решения зафиксированы 2026-07-11)

Обсуждено с пользователем; выбранные варианты помечены. Документ — источник
истины при имплементации; спорное — сюда, а не в код.

## Объём

Пять звуков: мотор игрока (pitch от скорости), тики светофора рядом,
удар при столкновении, сигнал фиксации нарушения, клаксон раздражённых NPC.

## Принятые решения

1. **Технология: чистый синтез Web Audio API** (осцилляторы + шум + огибающие).
   Ноль ассетов — проект полностью процедурный, и звук такой же. Звучание
   осознанно «аркадное».
2. **Позиционирование: только громкость по расстоянию**, без стереопанорамы.
   Затухание от машины игрока (не от центра экрана): `gain = clamp01(1 - d/R)²`,
   R — радиус слышимости источника.
3. **Клаксон NPC — два триггера** (только на игрока, NPC друг другу не сигналят —
   иначе какофония в любой пробке):
   - «блокировка»: NPC стоит, в его follow-конусе впереди — игрок, и так ≥3 с →
     гудок, повтор каждые ~4–6 с пока ситуация держится;
   - «подрезание»: игрок появился на траектории движущегося NPC и требуемое
     замедление резкое (v²/2d > ~4 м/с²) → одиночный гудок сразу, латч на эпизод.
4. **Светофор — как настоящий (сигналы для незрячих), привязан к фазе**:
   слышен только ближайший светофорный узел в радиусе ~25 м, фаза берётся для
   подъезда ИГРОКА (`approachOf`); красный/красно-жёлтый — редкие тики (~1 Гц),
   зелёный — частые (~6 Гц), жёлтый — молчит. Не на подъезде (в квадрате,
   удаляется) — тишина.

## Архитектура

Звук — строго UI-слой; движок остаётся беззвучным и тестируемым. Игровая
сторона лишь **порождает события**, аудио их **озвучивает**.

```
src/game/Traffic.ts  NpcVehicle детектит оба триггера клаксона (у него уже есть
                     follow-конус, где игрок = id -1); Traffic.consumeHonks()
                     → HonkEvent[] {x, y}. Детекция работает ВСЕГДА (не зависит
                     от включённости звука) — иначе поведение/тесты ветвятся.
                     НЕ добавлять вызовов rng (сломает детерминизм маршрутов
                     NPC и свип автопилота): джиттер повтора — из id
                     (напр. 4 + id % 3 с).
src/game/Round.ts    contact: boolean — «в этом шаге был откат hitsSolid»
                     + impactSpeed (скорость ДО обнуления, для громкости удара).
                     Свежие нарушения уже возвращает step().
src/ui/Sound.ts      НОВЫЙ. class Sound { ctx: AudioContext | null }.
                     init() — по клику «Поехали!» (жест в main.ts:71 заготовлен
                     под это). update(round, dt) каждый кадр: мотор, тики
                     светофора, съём consumeHonks()/contact/fresh violations.
                     Все триггеры по фронту (rising edge), не по уровню.
src/main.ts          создаёт Sound, зовёт update() после round.step();
                     кнопка 🔇 (btn-mute), состояние в localStorage
                     ('drive-town-sound', отдельный ключ — не трогаем схему
                     SaveState); suspend/resume по visibilitychange.
```

## Уточнения, принятые при имплементации

- **Очередь на светофоре не гудит**: «блокировка» подавляется, пока свет по
  курсу NPC не зелёный — стоять пришлось бы и без игрока (иначе NPC гудел бы
  игроку, легально ждущему красный). Таймер 3 с стартует заново с зелёного.
  Побочный эффект (принят): игрок, блокирующий NPC далеко от светофора,
  услышит гудки только в зелёные окна цикла.
- **«Подрезание» оценивается на фронте появления в конусе** (rising edge
  inCone): NPC при мгновенном торможении сбрасывает скорость в тот же тик,
  поэтому детекция идёт ДО step по скорости прошлого тика. Обычное сближение
  с медленным игроком не гудит: на входе в конус (22 м) v²/2d ≈ 2.3 < 4.
- **HonkEvent расширен до {x, y, kind, n}**: kind нужен палитре (длительность
  0.4/0.7 с), n — номер в серии «блокировки» (повторы длиннее, потолок 1.2 с).
- Тики светофора: слышны при подъезде d ∈ (0, 25) до стоп-линии; в квадрате
  и при удалении approachOf уже смотрит на следующий узел — тишина сама собой.

## Палитра (рецепты синтеза)

| Звук | Рецепт | Параметры |
|---|---|---|
| Мотор | 2 осциллятора (saw + tri, лёгкий detune) → lowpass → gain; работает постоянно, пока раунд идёт | f = 55 + 14·\|v\| Гц (холостой 55, ~180 на 9 м/с); громкость чуть растёт со скоростью |
| Тик светофора | короткий sine-бип 15 мс, 880 Гц (зелёный) / 660 Гц (красный) | период 0.16 с (зелёный) / 1.0 с (красный); громкость по расстоянию, R=25 м |
| Удар | burst белого шума 80–150 мс → lowpass 300 Гц + sine 60 Гц «бух» | громкость ∝ impactSpeed; повтор не чаще 0.3 с (контакт длится кадрами) |
| Нарушение | двухнотный нисходящий «бузер» (напр. 620→415 Гц, по 90 мс), square | без позиционирования — это UI-звук, синхронен с тостом |
| Клаксон | 2 square-осциллятора в малую секунду (~400+424 Гц) → gain-огибающая | «блокировка» 0.4 с, при повторах длиннее (раздражение растёт); «подрезание» 0.7 с; громкость по расстоянию, R=45 м |

Все включения/выключения — через короткие ramp'ы гейна (≥10 мс), чтобы не
щёлкало. Master-gain общий; mute = ramp в 0, контекст не закрываем.

## Жизненный цикл

- AudioContext создаётся лениво в обработчике «Поехали!» (autoplay-политика).
- Вкладка скрыта → `ctx.suspend()`, вернулась → `resume()`.
- Раунд finished → мотор глушится, разовые звуки дозвучивают.
- R/N (новый раунд/город) → сброс латчей и таймеров звука.

## Тесты (TDD-граница)

- **Игровая сторона — полноценный vitest**: таймер «блокировки» (стоит за
  игроком 3 с → событие, повтор через интервал, уехал → сброс), «подрезание»
  (латч на эпизод), `Round.contact/impactSpeed`. Прогнать `SWEEP=40 npm test`:
  детекция не должна менять поведение NPC ни на бит.
- **Sound.ts — тонкий**: чистые функции-маппинги (скорость→частота,
  фаза→период тиков, расстояние→громкость) выносятся и тестируются; сам граф
  WebAudio не тестируем (в vitest нет AudioContext), поэтому в нём — ноль логики.

## Не в объёме (бэклог)

- Стереопанорама/HRTF (решили: пока только громкость).
- Звук заноса ручника (шорох шин от lateralV) — просится, но отдельно.
- Моторы NPC, фоновый шум города.
- Регулятор громкости (пока только mute-кнопка).
