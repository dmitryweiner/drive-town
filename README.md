# Drive Town — городок ПДД

## [🚗Играть🚗](https://dmitryweiner.github.io/drive-town/)

Браузерная игра: вы ведёте жёлтую машинку по процедурно сгенерированному городку до точки назначения (флажок на миникарте), соблюдая правила дорожного движения. Добрались — город генерируется заново, начинается следующий уровень. Нарушения не прерывают игру, а копятся в счётчик: чем меньше нарушений и чем меньше время, тем больше очков в конце уровня.

Технологии: TypeScript + Vite, HTML5 Canvas. Физика машины, коллизии, правила, знаки и отрисовка заимствованы из [driving-trainer](../driving-trainer/) (кинематическая велосипедная модель, OBB/SAT-коллизии, израильские дорожные знаки).

## Спецификация игры

### Игровой цикл

1. Генерируется случайный город: нерегулярная сеть улиц (перекрёстки, T-образные, повороты, тупики), дома в кварталах, знаки, светофоры, односторонние улицы, зебры, зоны «30»/«40».
2. Игрок появляется на дороге; на миникарте в углу — весь город, позиция игрока (жёлтая точка) и цель (красный флажок). В шапке — расстояние до цели по прямой.
3. Игрок едет к цели. По городу ездят NPC: машины, мотоциклы, велосипеды — и ходят пешеходы (переходят по зебрам). NPC соблюдают правила и уступают по приоритету, в том числе игроку.
4. Нарушения ПДД фиксируются счётчиком (всплывашка + ❗ в шапке), игра продолжается.
5. Цель достигнута (машина в круге у флажка) — уровень завершён: показываются время, список нарушений и очки. По `N` / кнопке — новый город.

### Нарушения

| Тип | Условие |
| --- | --- |
| Проезд на запрещающий сигнал | пересечение стоп-линии на красный/красно-жёлтый; на жёлтый — только если можно было безопасно остановиться |
| Проезд знака «стоп» | нет полной остановки перед стоп-линией |
| Проезд ЖД-переезда | нет полной остановки перед стоп-линией переезда |
| Не уступил на кольце | въезд на круговое движение, когда по кольцу кто-то едет |
| Не уступил дорогу | выезд на перекрёсток под машину с приоритетом (главная дорога, помеха справа, встречный при левом повороте); приоритет оценивается на момент начала манёвра — встречный, появившийся, когда поворот уже идёт, не в счёт |
| Не пропустил пешехода | въезд на зебру, когда пешеход на проезжей части |
| Превышение скорости | выше лимита (город 50, зоны 30/40) дольше полсекунды |
| Выезд на встречную | движение по встречной полосе или против односторонней |
| Движение задним ходом | езда назад дольше ~0.7 с (короткий манёвр подравняться прощается) |
| Выезд с дороги | центр машины вне полотна |
| Столкновение | контакт с участником движения или домом (дома и NPC — твёрдые) |

Каждое нарушение считается один раз на эпизод (например, длинный выезд на встречную — одно нарушение, пока не вернулись на свою полосу).

### Очки

`score = max(0, round(1000 × par / max(время, par)) − 150 × нарушения)`,
где `par` — длина кратчайшего маршрута на «эталонной» скорости 7 м/с + 10 с запаса. Чистый и быстрый проезд — 1000 очков. Общий счёт копится между уровнями и хранится в `localStorage` (вместе с номером уровня и сидом текущего города — перезагрузка страницы не сбрасывает уровень). Кнопка 🗑 рядом с очками сбрасывает прогресс.

### Генерация города

- Нерегулярная осеориентированная сеть: решётка 8×6 с неравномерными кварталами (55–95 м), из которой случайно выброшена часть улиц — так получаются T-образные перекрёстки, повороты и тупики. Связность гарантирована остовом.
- ~15 % улиц — односторонние, целыми «улицами»: режим движения меняется только на перекрёстках, а не посреди дороги (и только там, где город остаётся строго связным: тупиков-ловушек не бывает).
- Перекрёстки: светофорные, «главная дорога» (знаки 309/301/302 на подъездах), нерегулируемые (помеха справа) и кольцевые (знак 303): островок посередине, движение вокруг против часовой, въезжающий уступает тем, кто уже на кольце.
- На длинных улицах — зебры со знаками; зоны ограничения: «30» у переходов, «40» на отдельных улицах.
- До двух ЖД-переездов (знак 129): рельсы поперёк дороги, перед стоп-линией обязательна полная остановка — как в driving-trainer. NPC тоже останавливаются.
- Дома в кварталах — сетка разноцветных зданий с проулками и дворами; дома твёрдые.
- Спавн и цель — в разных концах города (минимум 5 перекрёстков по маршруту); цель всегда достижима без нарушений (проверяется автопилотом в тестах).

### Физика

Кинематическая велосипедная модель из parking-game + два расширения:
- **мягкий руль** — угол передних колёс меняется с ограниченной скоростью (`steerRate`), машина не дёргается от коротких нажатий;
- **ручной тормоз** (`Space`) — блокирует задние колёса: машина тормозит и, если повёрнут руль, уходит в занос (боковое скольжение с ослабленным сцеплением). Та же физика добавлена в parking-game и driving-trainer.

### Управление

| Действие | Десктоп | Мобильный |
| --- | --- | --- |
| Газ | `W` / `↑` | кнопка **▲** |
| Тормоз / задний ход | `S` / `↓` | кнопка **▼** |
| Ручной тормоз (занос) | `Space` | кнопка **🅿** |
| Руль | `A`/`D`, `←`/`→` | кнопки **◀ ▶** |
| Новый город (после финиша) | `N` / `Enter` | кнопка в оверлее |
| Уровень заново | `R` | кнопка **↻** |

При открытии игра ждёт клика «Поехали!» на стартовом экране с памяткой управления (этот клик-жест инициализирует звук). Камера следует за машиной; масштаб — колесо мыши / щипок; кнопка **🧭** в шапке включает вращение карты по курсу машины (по умолчанию выключено). Текущая скорость — крупно под шапкой слева. Отладка: `http://localhost:5173/?seed=123` — фиксированный город.

### Звук

Полностью процедурный синтез Web Audio (без ассетов, дизайн — в SOUND.md): тихий мотор с высотой тона от скорости, шорох шин при заносе ручника, тики ближайшего светофора (частые на зелёный, редкие на красный — как сигналы для незрячих), «бух» столкновений, бузер фиксации нарушения и клаксоны раздражённых NPC — если стоять у них на пути дольше 3 секунд или подрезать. Громкость позиционных звуков падает с расстоянием. Кнопка **🔊/🔇** в шапке выключает звук (запоминается в `localStorage`).

## Структура проекта

```
src/
  main.ts               bootstrap, игровой цикл, уровни, localStorage
  game/
    Car.ts              физика машины (из driving-trainer)
    Collision.ts        OBB + SAT (из driving-trainer)
    types.ts            типы города и нарушений
    rng.ts              seeded ГСЧ (mulberry32)
    CityMap.ts          граф улиц: полосы, стоп-линии, повороты дугами,
                        светофорные фазы, зебры, маршруты (BFS), isOnRoad
    generate.ts         процедурный генератор уровня (город + спавн + цель)
    Rules.ts            монитор нарушений ПДД (счётчик, не «провал»)
    Traffic.ts          NPC: машины/мото/вело по случайным маршрутам
                        с уступанием, бронью перекрёстков и дистанцией;
                        пешеходы на зебрах; детекция гудков клаксона
    Round.ts            раунд: машина + трафик + правила + цель + очки
  ui/
    Input.ts            клавиатура + тач (из driving-trainer)
    Renderer.ts         отрисовка города, знаков, светофоров, машин
    Minimap.ts          миникарта с игроком и целью
    Hud.ts              шапка, тосты нарушений, оверлей финиша
    Sound.ts            процедурный звук Web Audio (см. SOUND.md)
tests/
  car/collision         физика и коллизии (из driving-trainer)
  citymap.test.ts       геометрия города
  generate.test.ts      инварианты генератора (связность, знаки, дома)
  rules.test.ts         каждое правило по отдельности
  traffic.test.ts       поведение NPC (светофор, стоп, приоритет, зебра)
  round.test.ts         раунд и формула очков
  sound.test.ts         чистые маппинги звука (частоты, периоды, затухание)
  completable.test.ts   сквозной: автопилот проходит уровни без нарушений
  helpers/autopilot.ts  «идеальный водитель» (pure pursuit + приоритеты)
```

## Разработка

Нужен Node.js 18+.

```bash
npm install
npm test               # vitest
SWEEP=40 npm test      # + широкий свип автопилота по 40 городам
npm run lint           # ESLint (`as`-касты запрещены)
npm run dev            # http://localhost:5173
npm run build          # tsc + сборка в ./docs (GitHub Pages)
npm run shot -- 7 --drive 4   # скриншоты города в ./shots (headless Chromium)
```

**TDD.** Тесты пишутся раньше кода; движковые модули (`game/`) покрыты юнит-тестами, разрешимость сгенерированных уровней проверяется автопилотом (`completable.test.ts`). Рендер и DOM — вручную.
